## Как открыть локально

Открой `index.html` в браузере. Данные сейчас лежат в `data/schedule.js`, поэтому страница открывается без сервера.

## Основной сценарий

1. Выбери тренера.
2. Дашборд покажет всё его расписание на неделю.
3. При необходимости сузь расписание по дню недели.
4. Затем выбери конкретную тренировку.

В дашборд выводятся тренер, тренировка, адрес и время. Округи и служебные строки отбрасываются при разборе данных.

## Как обновляются данные на GitHub

Workflow `.github/workflows/pages.yml` делает три вещи:

1. Скачивает CSV из Google Sheets.
2. Пересобирает `data/schedule.json` и `data/schedule.js`.
3. Публикует сайт через GitHub Pages.

Автообновление стоит раз в 6 часов, также есть ручной запуск через `workflow_dispatch`.

## Настройка GitHub Pages

В репозитории открой `Settings -> Pages` и выбери `GitHub Actions` как источник публикации.

## Формат, который дашборд понимает лучше всего

Лучший вариант листа:

| День | Время | Тренер | Группа | Зал | Комментарий |
| --- | --- | --- | --- | --- | --- |
| Понедельник | 15:00-16:00 | Имя тренера | Название группы | Зал 1 |  |

Если лист устроен как сетка, дашборд всё равно покажет сырые строки и попробует собрать занятия автоматически.
